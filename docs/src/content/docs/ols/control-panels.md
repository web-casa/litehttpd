## Overview

Several hosting control panels support OpenLiteSpeed, providing a GUI for managing virtual hosts, SSL certificates, PHP versions, and more. This page covers the most common integrations.

## CyberPanel

CyberPanel is the official control panel built for OpenLiteSpeed. It provides the most complete OLS integration available.

### Install CyberPanel + OLS

```bash
sh <(curl -s https://cyberpanel.net/install.sh)
```

During installation, select **OpenLiteSpeed** when prompted for the web server.

### Features

- Automatic OLS virtual host management
- One-click SSL via Let's Encrypt
- PHP version switching per site
- LiteSpeed Cache management
- Email server (Postfix + Dovecot)
- DNS server (PowerDNS)
- File manager, database manager, backup/restore
- Docker support
- Git deployment

### OLS Configuration

CyberPanel manages OLS configuration automatically. Configuration files are at:

- Server config: `/usr/local/lsws/conf/httpd_config.conf`
- VHost configs: `/usr/local/lsws/conf/vhosts/{domain}/vhconf.conf`

:::caution
Avoid editing OLS configuration files directly when using CyberPanel. Changes may be overwritten by the panel. Use the CyberPanel web interface or CLI instead.
:::

### CyberPanel CLI

```bash
# List websites
cyberpanel listWebsites

# Create website
cyberpanel createWebsite --domainName example.com --email admin@example.com \
  --package Default --owner admin

# Issue SSL
cyberpanel issueSSL --domainName example.com
```

### Using LiteHTTPD with CyberPanel

The LiteHTTPD module works alongside CyberPanel. Install it into the OLS modules directory, and `.htaccess` files placed by applications (WordPress, Laravel, etc.) will be processed automatically.

## aaPanel

aaPanel is a popular control panel that supports OLS via a plugin.

### Install aaPanel

```bash
# CentOS/AlmaLinux
URL=https://www.aapanel.com/script/install_7.0_en.sh && \
  if [ -f /usr/bin/curl ]; then curl -ksSO "$URL"; else wget --no-check-certificate -O install_7.0_en.sh "$URL"; fi && \
  bash install_7.0_en.sh aapanel
```

### Install OpenLiteSpeed Plugin

1. Log in to the aaPanel web interface.
2. Navigate to **App Store**.
3. Search for "OpenLiteSpeed" and install.
4. Follow the prompts to replace the default web server with OLS.

### Configuration

aaPanel manages OLS configuration through its web interface. Key paths:

- Server config: `/usr/local/lsws/conf/httpd_config.conf`
- VHost configs: generated by aaPanel under `/usr/local/lsws/conf/vhosts/`

## DirectAdmin

DirectAdmin supports OLS through the CustomBuild 2.0 system.

### Switch to OLS in DirectAdmin

```bash
cd /usr/local/directadmin/custombuild
./build set webserver openlitespeed
./build openlitespeed
./build rewrite_confs
```

### Features with DirectAdmin

- Automatic vhost configuration
- PHP version management via CustomBuild
- SSL certificate management
- Per-user resource isolation

### Switching Back to Apache

```bash
cd /usr/local/directadmin/custombuild
./build set webserver apache
./build apache
./build rewrite_confs
```

## OLS WebAdmin Console

OLS ships with its own web-based administration console, independent of any control panel.

### Access WebAdmin

The WebAdmin GUI runs on port 7080 by default:

```
https://your-server-ip:7080
```

### Set Admin Password

```bash
/usr/local/lsws/admin/misc/admpass.sh
```

### WebAdmin Features

- Server configuration (listeners, modules, tuning)
- Virtual host management
- External application setup (PHP, proxy backends)
- SSL certificate management
- Real-time server statistics
- Log viewer
- Graceful restart

### Security

Restrict WebAdmin access to trusted IPs:

```apacheconf
listener admin {
  address                 *:7080
  secure                  1

  accessControl {
    allow                 YOUR_IP_ADDRESS
    deny                  ALL
  }
}
```

## Troubleshooting

**Panel changes not taking effect:**
- Restart OLS after panel configuration changes: `systemctl restart lsws`
- Check for configuration syntax errors: `/usr/local/lsws/bin/lshttpd -t`

**Panel cannot connect to OLS:**
- Verify OLS is running: `systemctl status lsws`
- Check that the WebAdmin listener is active on port 7080

**Conflicts between panel and manual OLS edits:**
- Panels may overwrite manual configuration changes
- Use the panel's interface for all changes, or exclude specific vhosts from panel management
