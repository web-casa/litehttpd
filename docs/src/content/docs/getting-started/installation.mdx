## LiteHTTPD Installation

LiteHTTPD (Full) is the default and recommended edition. It includes the patched OLS binary and the htaccess module.

### Prerequisites

- OpenLiteSpeed 1.7.0 or later
- Linux x86_64 (ARM64 not yet supported)

<Tabs>
<TabItem label="RPM Repo (EL 9) -- recommended">

<Steps>
1. Add the LiteHTTPD RPM repository and install:
   ```bash
   curl -s https://rpms.litehttpd.com/setup.sh | bash
   dnf install openlitespeed-litehttpd
   ```

   This single RPM includes the patched OLS binary, the bundled htaccess module, and auto-configuration. Packages are GPG signed.

2. Restart OLS:
   ```bash
   systemctl restart lsws
   ```
</Steps>

</TabItem>
<TabItem label="Manual RPM download">

<Steps>
1. Download from [rpms.litehttpd.com](https://rpms.litehttpd.com/) or add the repository:

2. Install:
   ```bash
   dnf install ./openlitespeed-litehttpd-*.rpm
   ```

3. Restart OLS:
   ```bash
   systemctl restart lsws
   ```
</Steps>

</TabItem>
<TabItem label="Binary tarball (any distro)">

Distro-independent tarballs (glibc 2.28+, built like the official OLS tarball — extract and run `install.sh`). Download from [rpms.litehttpd.com](https://rpms.litehttpd.com/) or the GitHub release.

<Steps>
1. **Fresh install** (no OpenLiteSpeed yet) — installs the complete patched OLS + module:
   ```bash
   curl -LO https://rpms.litehttpd.com/openlitespeed-litehttpd-1.8.5-x86_64-linux.tgz
   tar xzf openlitespeed-litehttpd-1.8.5-x86_64-linux.tgz
   cd openlitespeed-litehttpd-1.8.5 && sudo ./install.sh
   ```

2. **Already running official OpenLiteSpeed** — pick an overlay:

   - **Full features** (overlay the patched binary — must match your installed OLS version; backs up the original):
     ```bash
     tar xzf litehttpd-ols-patch-1.8.5-x86_64-linux.tgz
     cd litehttpd-ols-patch-1.8.5 && sudo ./install.sh
     ```
   - **Module only** (works on *any* OLS version; no binary change — `RewriteRule`/`php_value` are parsed but not executed):
     ```bash
     tar xzf litehttpd-module-0.2.2-x86_64-linux.tgz
     cd litehttpd-module-0.2.2 && sudo ./install.sh
     ```

3. Restart OLS:
   ```bash
   /usr/local/lsws/bin/lswsctrl restart
   ```
</Steps>

</TabItem>
<TabItem label="From Source (Full)">

<Steps>
1. Install build dependencies:
   ```bash
   # AlmaLinux / Rocky / RHEL
   dnf install cmake gcc gcc-c++ libcrypt-devel

   # Ubuntu / Debian
   apt install cmake g++ libcrypt-dev
   ```

2. Clone OLS, apply patches, and build:
   ```bash
   git clone --branch v1.8.5 https://github.com/litespeedtech/openlitespeed.git
   cd openlitespeed
   patch -p1 < /path/to/patches/0001-lsiapi-phpconfig.patch
   patch -p1 < /path/to/patches/0002-lsiapi-rewrite.patch
   # Apply patches 0003, 0004
   bash build.sh
   ```

3. Build the module:
   ```bash
   cd /path/to/litehttpd
   cmake -B build -DCMAKE_BUILD_TYPE=Release
   cmake --build build --target litehttpd_htaccess
   ```

4. Install:
   ```bash
   cp build/litehttpd_htaccess.so /usr/local/lsws/modules/
   ```
</Steps>

</TabItem>
</Tabs>

## LiteHTTPD-Thin (Module Only)

For stock OLS without patches. Quick to set up but limited:

- RewriteRule directives are parsed but not executed
- `php_value`/`php_flag` directives are parsed but not passed to lsphp

<Steps>
1. Copy the module to OLS:
   ```bash
   cp litehttpd_htaccess.so /usr/local/lsws/modules/
   ```

2. Enable the module in OLS config:
   ```bash
   cat >> /usr/local/lsws/conf/httpd_config.conf <<'CONF'

   module litehttpd_htaccess {
       ls_enabled              1
   }
   CONF
   ```

3. Restart OLS:
   ```bash
   /usr/local/lsws/bin/lswsctrl restart
   ```
</Steps>

Consider upgrading to Full for production WordPress sites that rely on `.htaccess` redirects.
